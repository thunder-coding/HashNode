## 1.  TypeScript settings

Do you know that TypeScript has support for settings?

If not you can know now.

In the root directory of your TypeScript projects open a fresh new terminal. And type `tsc --init`. This command will create a `tsconfig.json`, which will have settings on how your TypeScript files will be compiled and where the output `*.JS` files will go. The file will have a lot of comments, which will help you configuring the file.

## 2.  Compile TypeScript in background

In your source folder, open a new terminal or use the old one. And run `tsc -w`.

This will run TypeScript in watch mode. Means TypeScript will check for any changes in `*.ts` files in your folder and compile them whenever it finds any changes.


## 3.  Run TypeScript in Node.js without Compiling TypeScript to JavaScript

Install `ts-node` npm package üì¶ by running `npm install -g ts-node`

Then run any existing `*.ts`file by running `ts-node <filename>`

> Note: ts-node must be used only for development purpose


## 4.  Tell TypeScript to ignore certain erorrs in files.

```
// @ts-ignore
document.getElementById("id").style = "";
```

Even though the above code has error, the TypeScript compiler will not complaint about it.
> Note: Use this command carefully üòâ

> This command does not ignore serious language errors such as

```
const const = "String"; // ‚ùå Error TS1134: Variable declaration expected.
```

This ts-ignore flag only ignores type-checking errors



## 5.  Use `as any` as a wildcard to avoid TypeScript scolding you for assigning incorrect types
